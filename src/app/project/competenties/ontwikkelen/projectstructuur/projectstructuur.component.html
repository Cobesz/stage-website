Projectstructuur

Situatie

Het nieuw te starten project “BBSdesk” gaat een software platform worden waarin verschillende apps terecht gaan komen. Denk aan: Kennisbank, Nieuws, callcentre crm, rooster etc. Al deze modules moeten onafhankelijk van elkaar kunnen opereren.


  • Taak

    Aan mij de taak om de structuur van het project op te zetten en te bepalen hoe de code ingezet wordt aan de voorkant van het project. Dit betreft een front-end in angular 4. Aan mij om een flexibele en duidelijk te begrijpen projectstructuur op te zetten. Uiteraard is het ook de bedoeling dat deze structuur ook na mijn stage nog gebruikt moet worden.


    Activiteiten

    Ik ben begonnen met het opdelen van de op dat moment bekende modules waar aan gewerkt ging worden. Dit betrof de Kennisbank- en Nieuwsmodule. Ook heb ik een gedeelde Module gemaakt onder de noemer “SharedModule”. Deze module bevat globale functies, styling en elementen die in het hele project nodig zijn. Denk bijvoorbeeld aan een footer of een header. In de root van het project staan de configuraties van het project. Hieronder is te zien hoe de eerste laag is opgesteld.




    Vervolgens heb ik de modules opgedeeld in services en componenten, dit met als doeleinde om de code overzichtelijk en duidelijk te houden. De componenten bevatten stuk voor stuk een html (element opbouw), scss(styling) en typescript (logica) bestand.Een voorbeeld van een component is faqComponent, het doel van dit component is om de detailweergave van een veelgestelde vraag te laten zien aan de eindgebruiker.




    Zo ziet het component er visueel uit.

    Resultaat

    Deze structuur heeft er voor gezorgd dat we als team meer duidelijkheid hebben over het te bouwen project. Ook zorgt het voor de modulaire opbouw waar Bol & Boosch naar zocht, zodat de flexibiliteit van het product gewaarborgd kan worden.
