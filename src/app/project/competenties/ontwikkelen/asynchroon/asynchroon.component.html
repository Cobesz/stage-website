Asynchroon Programmeren

Situatie

Naar mijn mening liep code altijd van links naar rechts, van boven naar beneden. Een computer leest code als een boek. Tijdens het programmeren van de front-end van BBSdesk liep ik hier tegenaan.


  • Taak

    Uitzoeken waarom er allerlei errors op de console van het project terecht kwamen, waarom stukken code niet werkte en hoe dit kan. Deze errors verschenen omdat ik een api call deed naar de back-end van het project en deze vervolgens op de front-end wilde laten zien.


    Activiteiten

    Mede door hulp van mijn stagebegeleider en een collega ben ik mij in gaan lezen in callbacks en promises. Ook ben ik op youtube gaan zoeken naar mensen die het me uit konden leggen op een begrijpbare manier. Een aantal kanalen hebben mij flink geholpen, namelijk:


    • https://www.youtube.com/channel/UCO1cgjhGzsSYb1rsB4bFe4Q
    • https://www.youtube.com/user/shiffman

    Blijkbaar kan je code op elkaar laten wachten, door bijvoorbeeld te “beloven” dat er een resultaat aan komt waar je later iets mee kan doen. Dit stukje code heb ik vervolgens geschreven.




    Resultaat

    Hierdoor kreeg ik een heel ander idee over hoe je code kan lezen en ben ik tot de ontdekking gekomen dat je code op elkaar kan laten wachten tot er een resultaat is wat er verwacht wordt. Deze resultaten kan je afvangen, of als hier een fout in is op reageren. Het heeft er toe geleid dat ik code makkelijker kan lezen en beter in kan stappen in toekomstige projecten.


    Reflectie

    Dit punt was best moeilijk voor mij om door te krijgen, achteraf gezien zou ik sneller naar goede uitleg gezocht hebben in plaats van zelf aan de slag gaan zonder enig idee van hoe het werkt. Ik ben wel van mening dat ik nu beter snap hoe het niet werkt en vooral daarom de achterliggende gedachte goed snap.
