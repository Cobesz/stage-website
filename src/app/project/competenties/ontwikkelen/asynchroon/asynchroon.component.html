Asynchroon Programmeren

Situatie

Naar mijn mening liep code altijd van links naar rechts, van boven naar beneden. Een computer leest code als een boek. Tijdens het programmeren van de front-end van BBSdesk liep ik hier tegenaan.


  • Taak

    Uitzoeken waarom er allerlei errors op de console van het project terecht kwamen, waarom stukken code niet werkte en hoe dit kan. Deze errors verschenen omdat ik een api call deed naar de back-end van het project en deze vervolgens op de front-end wilde laten zien.


    Activiteiten

    Mede door hulp van mijn stagebegeleider en een collega ben ik mij in gaan lezen in callbacks en promises. Ook ben ik op youtube gaan zoeken naar mensen die het me uit konden leggen op een begrijpbare manier. Een aantal kanalen hebben mij flink geholpen, namelijk:


    • https://www.youtube.com/channel/UCO1cgjhGzsSYb1rsB4bFe4Q
    • https://www.youtube.com/user/shiffman

    Vervolgens heb ik de modules opgedeeld in services en componenten, dit met als doeleinde om de code overzichtelijk en duidelijk te houden. De componenten bevatten stuk voor stuk een html (element opbouw), scss(styling) en typescript (logica) bestand.Een voorbeeld van een component is faqComponent, het doel van dit component is om de detailweergave van een veelgestelde vraag te laten zien aan de eindgebruiker.




    Zo ziet het component er visueel uit.

    Resultaat

    Deze structuur heeft er voor gezorgd dat we als team meer duidelijkheid hebben over het te bouwen project. Ook zorgt het voor de modulaire opbouw waar Bol & Boosch naar zocht, zodat de flexibiliteit van het product gewaarborgd kan worden.

    Het is een structuur geworden die tot de dag van vandaag nog steeds gebruikt wordt met weinig tot geen klachten.
